# Glossary

Learn some of the basic terms and primitives of `pacwich` here.

## Terms

#### Project

A **project** is typically the same as the root of your git repository for your monorepo. This is where your top-level `package.json` 
file is located and where you've defined workspaces for your package manager, whether via the `"workspaces"` field in
package.json (for Bun or npm) or `pnpm-workspace.yaml` (for pnpm).

`pacwich` recognizes a project by a root `package.json` living alongside a lockfile (`package-lock.json`, `bun.lock`, `pnpm-lock.yaml`), since the 
lockfile is the best source of truth for workspace-related data, so a project and its workspace data aren't recognized or current until you run your install command.

#### Workspace

A **workspace** is simply a nested package within your project that has its own `package.json` file and is matched by the workspaces
you declare at the root of your project with your package manager's config. 

`pacwich` simply builds its features to enrich and work with plain workspaces.

#### Script

In the context of `pacwich`, a **script** generally refers to an entry in the `"scripts"` field in a workspace's `package.json` file by default. 

However, pacwich also supports running ["inline scripts"](/concepts/inline-scripts), one-off shell commands executed from a respective workspace's directory.

#### Inputs

**Inputs** are values considered to affect the result of a script or state of a workspace.

These include git-trackable files, other workspaces, and external dependencies (npm packages).

By default, all input files are simply the git-trackable files found in a workspace's directory,
workspace inputs are explicit [workspace dependencies](/concepts/workspace-dependencies),
and external dependencies are tracked via lockfiles.

These are currently used to determine [affected workspaces](/concepts/affected) (where features
generally accept options for ignoring some inputs), and input files specifically are used by the 
[`verify` feature](/cli/commands#verify) to check for missing workspace dependencies by analyzing imports/exports
in JS/TS input files.
