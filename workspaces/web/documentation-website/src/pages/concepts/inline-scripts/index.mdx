# Concept:  Inline Scripts

## Overview

Normally, a ["script"](/concepts/glossary#script) in pacwich references one of the entries found in a workspace's `"scripts"` field in its `package.json` file.

However, it's possible to run a one-off shell command from a workspace's directory, an "inline script", such as via `pacwich run "echo 'hello'" --inline` in the CLI.

## Using the Bun Shell

Normally, inline scripts are executed using your system shell. In POSIX systems (Linux, macOS, etc.), `sh -c` is called, while in Windows, `cmd /d /s /c` is called.

You can also use the [Bun shell](https://bun.com/docs/runtime/shell), a bash-like, cross-platform shell, the same as the utility `import { $ } from 'bun'` by passing `--shell=bun`,
if the `bun` command is available.

You can also set the [root](/config/project) <code>{ENV_VARS_METADATA.scriptShellDefault.projectConfigDefaultsKey}</code> or [environment variable](/config/env-vars) <code>{ENV_VARS_METADATA.scriptShellDefault.envVarName}</code> to `"bun"` to use the Bun shell by default.

## Script Name

Scripts from `package.json` are named by the key in the `"scripts"` field. 

This is used in output logs and [workspace script metadata](/concepts/workspace-script-metadata).

Inline scripts, however, have no name by default, but you can provide a name.
