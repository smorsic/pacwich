# Concept:  Workspace Script Metadata

## Overview

When running scripts via the API or CLI, metadata is available via either environment variables within your scripts or interpolation
in an [inline script](/concepts/inline-scripts) or appended args.

The environment variables allow scripts to introspect metadata about the project, workspace, and script info to get
context about its execution by `pacwich`.

This same metadata can be referenced within inline script commands and/or appended script args.

## Examples

#### Interpolation via CLI:
In this command, all workspaces will run the same `my-script.ts` file found at the project root, thanks to using `<projectPath>` to interpolate the project root's path, 
and the workspace name and path will be passed to the script as arguments.

<SyntaxHighlighter language="bash">
{`pacwich run "bun <projectPath>/my-script.ts" --inline \\
\ \ --inline-name="my-script-name" \\
\ \ --args="<workspaceName> <workspacePath>"`}
</SyntaxHighlighter>

The script itself could use metadata from the environment variables to dynamically change
its behavior based on the workspace context passed to it:

#### Interpolation via API:
The same as the above CLI example using the TypeScript API:
<SyntaxHighlighter language="typescript">
{`
import { createFileSystemProject } from 'pacwich';

const project = createFileSystemProject();

project.runScriptAcrossWorkspaces({
${"  "}script: "bun <projectPath>/my-script.ts",
${"  "}args: "<workspaceName> <workspacePath>",
${"  "}inline: { scriptName: "my-script-name" },
});`}
</SyntaxHighlighter>

#### Accessing in a script via environment variables

You can access metadata from within a script that was invoked via `pacwich` via 
environment variables directly or the `getWorkspaceScriptMetadata` helper.

<SyntaxHighlighter language="typescript">
{`
import { getWorkspaceScriptMetadata } from 'pacwich/script';

// Use the helper within a script that was invoked via pacwich
const projectPath = getWorkspaceScriptMetadata("projectPath");
const projectName = getWorkspaceScriptMetadata("projectName");
const workspaceName = getWorkspaceScriptMetadata("workspaceName");
const workspacePath = getWorkspaceScriptMetadata("workspacePath");
const workspaceRelativePath = getWorkspaceScriptMetadata("workspaceRelativePath");
const scriptName = getWorkspaceScriptMetadata("scriptName");

// Access via plain environment variables (same values as above)
console.log(process.env.${getWorkspaceScriptMetadataConfig("projectPath").envVarName});
console.log(process.env.${getWorkspaceScriptMetadataConfig("projectName").envVarName});
console.log(process.env.${getWorkspaceScriptMetadataConfig("workspaceName").envVarName});
console.log(process.env.${getWorkspaceScriptMetadataConfig("workspacePath").envVarName});
console.log(process.env.${getWorkspaceScriptMetadataConfig("workspaceRelativePath").envVarName});
console.log(process.env.${getWorkspaceScriptMetadataConfig("scriptName").envVarName});
`.trim()}
</SyntaxHighlighter>

## Available Metadata

#### Project Path
The absolute path to the project root.<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("projectPath").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("projectPath").envVarName}</code>

#### Project Name
The name of the project (as in its `package.json` `"name"` field).<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("projectName").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("projectName").envVarName}</code>

#### Workspace Name
The name of the workspace (as in its `package.json` `"name"` field).<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("workspaceName").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("workspaceName").envVarName}</code>

#### Workspace Path
The absolute path to the workspace.<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("workspacePath").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("workspacePath").envVarName}</code>

#### Workspace Relative Path
The relative path to the workspace from the project root.<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("workspaceRelativePath").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("workspaceRelativePath").envVarName}</code>

#### Script Name
The name of the script. This is an empty string when running as an inline script, unless a script name is [explicitly provided](/concepts/inline-scripts#script-name).<br/>

In inline scripts and args: <code>{getWorkspaceScriptMetadataConfig("scriptName").inlineName}</code><br/>
Environment variable: <code>{getWorkspaceScriptMetadataConfig("scriptName").envVarName}</code>
