# Configuration: Project Config File
 
The optional project configuration is the general configuration for a [project](/concepts/glossary#project) at a file such as <code>{createConfigLocationPath("jsonFile", PROJECT_CONFIG_FILE_NAME, PROJECT_CONFIG_PACKAGE_JSON_KEY)}</code> or in its `package.json` file.

## Type Definition

<SyntaxHighlighter language="typescript">
  {PROJECT_CONFIG_TYPE}
</SyntaxHighlighter>

### Defaults

The values in <code>{CONFIG_DEFAULTS_KEY}</code> can also be set by [environment variables](/config/env-vars). Configuration values take precedence over environment variables.

* <code>{ENV_VARS_METADATA.cliScriptOutputStyleDefault.projectConfigDefaultsKey}</code>: The [output style](/cli/commands#run-script) to use for workspace scripts. The default is "grouped" when on a TTY, which always falls back to "prefixed" on non-TTY terminals. Otherwise, it is "prefixed". Other options are "plain" and "none".
* <code>{ENV_VARS_METADATA.parallelMaxDefault.projectConfigDefaultsKey}</code>: The [parallel max value](/concepts/parallel-scripts#parallel-max-value)
* <code>{ENV_VARS_METADATA.scriptShellDefault.projectConfigDefaultsKey}</code>: The [shell option for inline scripts](/concepts/inline-scripts#shell)
* <code>{ENV_VARS_METADATA.includeRootWorkspaceDefault.projectConfigDefaultsKey}</code>: Whether to include the [root workspace](/concepts/root-workspace) in the project's workspaces list
* <code>{ENV_VARS_METADATA.affectedBaseRefDefault.projectConfigDefaultsKey}</code>: The default git base ref for [affected workspaces](/concepts/affected) resolution
  (this is <code>main</code> when not overridden)
* <code>{ENV_VARS_METADATA.suppressWarningsDefault.projectConfigDefaultsKey}</code>: Suppress warning logs by `WarningId`. See the [warnings page](/config/warnings) for a full list and other means of setting suppression.

### Workspace Pattern Configs

You can set workspace configs in bulk by using workspace patterns in `workspacePatternConfigs`.

See the [workspace pattern configs](/config/workspace-pattern-configs) page for more information.
